Описание

Что строим

Цель проекта

Running Coach - локальная система персонального бегового тренера и нутрициолога. Сначала делаем переносимый Telegram MVP на файлах, затем добавляем распознавание, аналитику, dashboard, питание и инфраструктуру.

Главный принцип

Не строить тяжелую платформу до первого полезного результата. Данные лежат локально, форматы прозрачные, LLM подключаются через OpenAI-compatible API.

Переносимость

Код и документация должны быть понятны Claude и Qwen без зависимости от Codex: промпты, схемы, инструкции и команды запуска лежат прямо в репозитории.

Техническое задание

Краткое ТЗ

MVP

  • Один Telegram-бот для бега.
  • Профиль спортсмена в data/profile.yaml.
  • Прием фото тренировок и сохранение в data/workouts/<date>/raw.
  • План дня и недели в JSON/Markdown.
  • Команды /today, /week, /profile.
  • QA по бегу через Claude/Qwen с коротким контекстом.

Ограничения первой версии

  • Без Postgres, Redis, Qdrant и Minio.
  • Без обязательного VDS-деплоя.
  • Без nutrition-бота в первом запуске.
  • Все исходные данные хранятся локально.

Переносимость

  • Обычный Python-код без зависимости от Codex.
  • LLM через base_url, api_key, model.
  • Промпты и JSON-схемы лежат отдельными файлами.
  • Инструкции описаны в AGENTS.md и CLAUDE.md.
